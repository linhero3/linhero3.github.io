Nginx 简单入门

发表于 2018-04-09 | 分类于 中间件 |

1. nginx windows 安装

1.1 下载

nginx download

1.2 启动

nginx -c nginx/nginx.conf

-c 指定从配置文件启动

1.3 访问nginx.conf配置的地址，显示nginx首页

Welcome to nginx!

2. nginx 常用命令

nginx -s command

-s(signal)：向主进程发送信号

  • nginx -c conf/nginx.conf 启动
  • reload 重新加载配置文件
  • quit 优雅关闭
  • stop 快速关闭
  • reopen 重新打开日志文件

3. nginx 常见用途

  • 静态HTTP服务器
  • 反向代理服务器
  • 负载均衡
  • 虚拟主机

3.1. 静态HTTP服务器

nginx是一个http服务器，客户端可以通过http协议访问nginx静态服务器上的静态资源。

server {
    listen 8081;
    server_name localhost;
}

location / {
    root  /staticResource;
    index index.html
}
  • 访问localhost:8081，会默认访问staticResource下的index.html。
  • js/css/jar等各种静态资源，均可通过http协议获取。

    http://localhost:8081/org.apache.util-1.0.0.jar
    http://localhost:8081/js/index.js

3.2. 反向代理服务器

1）代理服务器

客户端在发送请求时，不会直接发送给目的主机，而是先发给代理服务器。代理服务器收到请求后，向目的主机发出，并接收主机返回的数据，存放在代理服务器硬盘，再发送给客户机。

代理服务器用途：

  1. 缓存，提高访问速度
  2. 防火墙，过滤不安全信息
  3. 梯子，通过代理服务器访问不能访问的目标站点

2）正向代理与反向代理

  • 正向代理：客户端代理，为客户端收发请求，真实客户端对服务器不可见；如用于翻墙的代理服务器。

  • 反向代理：服务器代理，为服务器收发请求，真实服务器对客户端不可见；如CDN。

下面通过访问 www.youtube.com 来解释正向代理与反向代理。

1）自己搭梯子或者找代理服务器。

2）系统设置代理，windows在连接-局域网设置-代理服务器中配置代理服务器地址及端口。如 proxy.xxx.com 8080

此时，访问 youtube.com，请求首先会发送给代理服务器（正向代理），代理服务器再将请求发送给youtube.com（反向代理）,并接收youtube的响应放在代理服务器上，之后你从代理服务器拿走响应，返回给自己浏览器。

代理服务器访问youtube.com，即是访问反向代理服务器。反向代理服务器通过负载均衡等策略从真实的服务器获取响应（从静态代理服务器返回js/css等静态资源，从动态服务器返回后台数据）。

3）nginx反向代理

 server {  
    listen       80;                                                         
    server_name  localhost;                                               

    # 反向代理服务器
    location / {
        proxy_pass http://localhost:8080;
    }

    # 图片静态服务器
    location ~ \.(gif|jpg|png)$ {
        root F:\staticResource;
    }
}

3.3 负载均衡

nginx反向代理可以将客户端的请求分发给多个服务端，实现服务端的负载均衡。

upstream server-list {
    server 192.168.20.1:8080;
    server 192.168.20.2:8080;
}
server {
    listen 80;
    server_name localhost;
    location / {
        proxy_pass http://server-list;
    }
}

负载均衡策略：

  1. RR
    每个请求按时间顺序逐一分配到不同的后端服务器，如果后端服务器down掉，能自动剔除；如上面server-list配置。

  2. 权重

    upstream server-list {
    server 192.168.20.1:8080 weight=9;
    server 192.168.20.2:8080 weight=1;
}

  3. ip_hash
    上面两种负载均衡的策略会有一个问题，一个用户的多次请求可能会在不同的服务器中，这时候session如果没有在各个服务器之前同步，会造成鉴权等一系列问题。
    ip-hash负载均衡策略会按照请求的ip hash来分配服务器，所以一个用户固定访问一台服务器。

    upstream server-list {
    ip_hash;
    server 192.168.20.1:8080;
    server 192.168.20.2:8080;
}

  4. fair（第三方）
    按后端服务器的响应时间来分配请求，响应时间短的优先分配。

    upstream server-list { 
    fair; 
    server 192.168.20.1:8080;
    server 192.168.20.2:8080;
}

  5. url_hash（第三方）
    按访问url的hash结果来分配请求，使每个url定向到同一个后端服务器，后端服务器为缓存时比较有效。 在upstream中加入hash语句，server语句中不能写入weight等其他的参数，hash_method是使用的hash算法

    upstream backend { 
    hash $request_uri; 
    hash_method crc32; 
    server localhost:8080;
    server localhost:8081;
}

3.4 虚拟主机

有些小网站需要节省成本，可以把多个网站部署在同一个服务器上，分别对应同一个ip的不同端口。两个网站访问互不影响，就像访问两个服务器一样，所以叫虚拟主机;虚拟主机的原理是通过HTTP请求头中的Host是否匹配server_name来实现的。

server {
    listen 80;
    server_name www.a.com;
    location / {
        proxy_pass http://localhost:8080;
    }
}

server {
    listen 80;
    server_name www.b.com;
    location / {
        proxy_pass http://localhost:8081;
    }
}
